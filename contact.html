WAIT!!! - Before you send that Email

If you are trying to:

  • Fix a winning IOCCC entry - (see FAQ 5.2)
  • Update winning author information - (see FAQ 5.5)
  • Correct a web site problem - (see FAQ 5.4 or FAQ 5.6)

Instead of sending us Email, Please consider using the GitHub pull request process against the master branch of the ioccc-src/winner repo.

If you are trying to:

  • Report a bug in an IOCCC winning entry - (see FAQ 5.1)
  • Report an IOCCC web site problem - (see FAQ 5.3)
  • Report a broken or wrong web link - (see FAQ 5.6)

Instead first look at the IOCCC issues to see if the problem has already been reported. If it has been reported, feel free to add a comment to the issue. If you do not find an it has been reported, then fee free to open a new IOCCC issue.

See the above FAQ entries for more details.

Try mastodon

You may wish to follow us IOCCC on Mastodon as we use that platform to announce things such as the opening of a new IOCCC as well as winning entries of the IOCCC.

We no longer tweet (or whatever someone with poor impulse control calls posting messages on their platform), preferring to reside in the fediverse instead.

There is help with Mastodon available for those who might wish to get started with Mastodon.

While there are many fine Mastodon server instances out there, the IOCCC judges use fosstodon.org where lots of fun folk hang out, and topics like science, technology, free and open source have a good home. The fosstodon.oeg folk made the IOCCC feel welcome. And while using Mastodon is free, the IOCCC judges are pleased to support the Fosstodon Hub.
