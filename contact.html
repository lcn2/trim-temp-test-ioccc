How to contact the IOCCC

✓

WAIT!!! - Before you send that Email

If you are trying to:

  • Fix a winning IOCCC entry - (see FAQ 5.2)
  • Update winning author information - (see FAQ 5.5)
  • Correct a web site problem - (see FAQ 5.4 or FAQ 5.6)

Instead of sending us Email, Please consider using the GitHub pull request process against the master branch of the ioccc-src/winner repo.

If you are trying to:

  • Report a bug in an IOCCC winning entry - (see FAQ 5.1)
  • Report an IOCCC web site problem - (see FAQ 5.3)
  • Report a broken or wrong web link - (see FAQ 5.6)

Instead first look at the IOCCC issues to see if the problem has already been reported. If it has been reported, feel free to add a comment to the issue. If you do not find an it has been reported, then feel free to open a new IOCCC issue.

If you are trying to:

  • Comment or make a suggestion on IOCCC rules - (see FAQ 0.5)
  • Comment or make a suggestion on IOCCC guidelines - (see FAQ 0.5)
  • Comment or make a suggestion on IOCCC tools - (see FAQ 0.5)

Those FAQ entries will help guide you on how to make public comments and suggestion in the relevant places.

See the above FAQ entries for more details.

Try Mastodon

Please follow the IOCCC on Mastodon

We use Mastodon to make minor announcements, in addition announcing major events such as the opening of a new IOCCC, or who won.

See “FAQ 6.12: What is Mastodon and why does IOCCC use it?” for more information on Mastodon.
